Plan: 
Build a physical properties tool in python using the IDAES pure component Helmholtz EOS

Goal: 
Create a python package that calculates properties like entropy, density, heat capacity from the component, pressure, and temperature.
The tool should be available as a python function or as a CLI tool. 

Design decisions:
1. performing calculations in the IDAES helmholtz EOS requires pyomo, and idaes-pse
2. CLI framework will be argparse
3. package layout will be src/ layout with pyproject.toml
4. testing will be done with pytest

References:
1. I have created an example_property_calculation.py file to show how to use the IDAES framework.
2. The IDAES helmholtz eos documentation (https://idaes-pse.readthedocs.io/en/stable/reference_guides/model_libraries/generic/property_models/helmholtz.html) lists what functions, variables, and expressions are available to calculate.

Features:
1. python function with input component, pressure, temperature and calculates an available property
2. python function with input component, pressure, temperature, and outputs a dataframe with a subset (or all) of available properties
3. ability to generate dataframes of isobaric or isothermal properties by specifying a pressure or temperature range (and fixing the other) and calculating a subset (or all) available properties
4. CLI interface for feature 1
5. CLI interface for feature 2, prints dataframe to terminal
6. CLI interface for feature 3, prints dataframe to terminal

Notes:
Feature 2 is a generalization of feature 1 where the subset of output properties has only a single item.
Feature 3 is a generalization of feature 2 where the iterated state variable has 1 value.
Example output of feature 3 (isothermal properties):
Pressure & Temperature & Density & Enthalpy  \\
1        & 298         & 105     & 25.1      \\
3        & 298         & 118     & 23.4      \\
5        & 298         & 131     & 22.2      \\

Anticipated issues:
1. need extensive input validation
2. IDAES calculations require IPOPT, need exhaustive error handling in case IPOPT fails

Steps:
1. Project Setup & Initialization: Initialize `pyproject.toml` and `src/` layout. Implement a solver check for `ipopt` and set up Python logging.
2. Core Property Engine: Generalize `example_property_calculation.py` into a core module using P-H `StateVars`. Implement component validation using the IDAES helper.
3. Input Validation & Feature 1: Implement T-P input validation using Pyomo's units library. Implement single-point property calculation and write initial tests.
4. Multi-Property Output (Feature 2): Implement retrieval of multiple property expressions (handling phase-indexed expressions) and returning a 1D DataFrame including `phase_id`.
5. Batch Range Calculations (Feature 3): Implement an optimized range calculation using multiple state blocks in a single `ConcreteModel`.
6. Testing & Validation: Add comprehensive tests for Feature 2 and the batching logic in Feature 3.
7. CLI Implementation: Create the `argparse` CLI wrapper for all features.
8. Documentation & Finalization: Finalize the README with usage examples and ensure the package is installable.

Technical Implementation Notes:
1. Add a check for solver step in the initialization to ensure IPOPT is on path.
2. The tool will only support T-P input pairs and use the P-H StateVars.
3. The IDAES framework includes a helper function that lists supported components, add a validation step that ensures the desired component is in that list.
4. The output dataframe should include the phase_id so users know which state the properties represent, note that only some property expressions are indexed by phase.
5. Step 5 should implement a batching approach with state blocks.
6. Remove print statements and use python logging.
7. Pyomo has a library for dealing with units, implement unit checking with that before trying pydantic or pint.
