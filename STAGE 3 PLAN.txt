Plan:
Three focused extensions: saturation curve overlay, CSV export for DataFrame outputs, and simplified plot output file handling.

Goals:
1. Let users visualize where a property sweep crosses the saturation boundary by overlaying the saturation curve on plots.
2. Let users export any DataFrame-returning command (multi, range) to CSV instead of printing to terminal.
3. Remove the redundant --format flag on the plot command by inferring the file format from the --output filename extension.

Design Decisions:
1. Saturation overlay
   - Expose as `--saturation` CLI flag and `saturation=False` kwarg on plot_property()
   - Only meaningful when the y-axis property is pressure or when x-axis is temperature — skip silently with a warning if plot axes don't make sense (e.g. isobaric enth_mol sweep)
   - Generate the curve by sweeping temperature from triple-point to critical-point and calling a saturation property (pressure_sat or temperature_sat, both already in available_properties.csv)
   - Render as a dashed black line with label "saturation"
   - Critical point marked with a distinct marker (e.g. black X) when in view
2. CSV export
   - Add `--output` flag to `multi` and `range` subcommands
   - If --output is given, write DataFrame to CSV and print path; otherwise keep current behavior (print to terminal)
   - Infer CSV format from .csv extension; support .json as alternative (via DataFrame.to_json)
   - No change to the Python API — DataFrame.to_csv() is already trivial for users
3. Plot output format inference
   - Deprecate/remove `--format` CLI flag on `plot` subcommand
   - If --output has a recognized extension (.png, .svg, .pdf), use that
   - If --output has no extension or an unknown extension, error with a clear message
   - Default filename (when --output is omitted) still uses .png
   - Python API: keep `fmt` parameter but also infer from `save_path` extension if `fmt` is not explicitly set. Explicit `fmt` wins if both are provided.

Features:
1. `--saturation` flag on `idaes-props plot` and `saturation` kwarg on `plot_property()` that overlays the component's saturation curve.
2. `--output <file>` flag on `idaes-props multi` and `range` subcommands that writes the DataFrame to CSV or JSON based on extension.
3. Plot CLI infers output format from filename extension; `--format` is removed or deprecated.

Anticipated issues/notes:
1. Saturation curve range: need to query the component's triple-point and critical-point temperatures. These are available as `temperature_sat` at the triple point and `temperature_crit` / `pressure_crit`. Fall back to a reasonable default range if the properties are missing.
2. Saturation curve axes: only overlay when the plot axes match — i.e. the x-axis is temperature or pressure, and the y-axis is the other. For property-vs-T or property-vs-P plots where y is not P/T, emit a warning and skip.
3. CSV precision: pandas default to_csv() uses repr-level precision, which is usually fine. Don't force a specific float format — users can post-process if needed.
4. Multi-component saturation overlays: if plotting multiple components, draw a saturation curve per component in the matching color. Label each.
5. --format removal: this is a breaking CLI change for anyone using --format explicitly. Remove cleanly (not a deprecation warning) since the project is pre-1.0.
6. Extension parsing: use `os.path.splitext` and a whitelist of supported extensions. Lowercase the extension for comparison.
